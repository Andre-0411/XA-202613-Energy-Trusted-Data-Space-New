## 二、API 对接分析

### ✅ API Service 层结构规范

**架构**:
- `frontend/src/api/request.ts`: Axios 实例配置（baseURL, 拦截器）
- `frontend/src/api/*.ts`: 33 个 API service 文件，按模块划分
- `frontend/src/types/api.ts`: API 响应类型定义（`ApiResponse<T>`）
- 使用 `@tanstack/react-query`: React Query 数据缓存层

**示例**（来自 `auth.ts`）:
```typescript
import request from './request';
import type { ApiResponse, LoginRequest, LoginResponse } from '@/types/api';

export async function login(data: LoginRequest): Promise<LoginResponse> {
  const res = await request.post<ApiResponse<LoginResponse>>('/auth/login', data);
  return res.data.data;
}
```

**优点**:
- ✅ 类型导入清晰（`import type { ApiResponse }`）
- ✅ 返回类型明确（`Promise<LoginResponse>`）
- ✅ Axios 拦截器统一处理错误
- ✅ React Query 自动缓存和重试

## 三、潜在问题

### ⚠️ 1. API 响应字段名称不一致（已修复）

**历史问题**:
- 后端 `AgentConfig.to_dict()` 返回 `usage_count`，前端期望 `total_queries`
- 后端缺少 `avg_response_time` 字段，前端调用 `.toFixed(0)` 导致 TypeError

**修复**: 已修改 `backend/app/models/agent_model.py`，添加 `total_queries` 和 `avg_response_time` 字段

**现状**: 前后端字段兼容性问题已解决

---

### ⚠️ 2. 部分 API 未对接真实后端（Mock 数据残留）

**遗留问题**:
- 少量页面可能仍使用硬编码 Mock 数据（如 `MOCK_DATASETS`, `MOCK_PARTIES`）
- 需逐页面检查是否调用真实 API service

**修复进展**:
- `ComputeCreatePage`: 已移除 `MOCK_PARTIES/MOCK_DATASETS`，对接真实 API
- `DataMarketPage`: 已移除硬编码统计，数据驱动图表
- `SecurityDidPage`: 从 `listOrganizations` API 派生 DID 数据

**建议**: 继续检查剩余 ~48 个页面的 Mock 数据残留

---

### ⚠️ 3. 错误处理不够统一

**问题**: API service 层 Axios 拦截器处理错误，但前端页面未统一显示错误 Toast

**建议**:

# 后端服务层内存存储审查报告

**生成时间**: 2026-05-22 22:50
**审查范围**: `backend/app/services/*.py`
**发现问题**: 30+ 内存存储变量

---

## 一、严重问题（P0 - 必须修复）

### 1. alert_service.py - 告警系统完全内存化
```python
_alerts: Dict[str, dict] = {}                 # 告警记录
_alert_rules: Dict[str, dict] = {}            # 告警规则
_alert_history: List[dict] = []               # 告警历史
_notification_channels: Dict[str, dict] = {}  # 通知渠道
_silence_rules: Dict[str, dict] = {}          # 静默规则
_inhibition_rules: Dict[str, dict] = {}       # 抑制规则
```
**影响**: 服务重启后所有告警数据丢失，无法追溯历史告警，多实例部署时告警不一致

**修复方案**:
- 创建 `Alert`, `AlertRule`, `AlertHistory`, `NotificationChannel`, `SilenceRule`, `InhibitionRule` 数据库模型
- 所有 CRUD 操作改为 SQLAlchemy 异步查询
- 参考 `monitor_service.py` 的 `MonitorAlert` 模型实现

---

### 2. agent_service.py - AI 对话历史内存存储
```python
_conversation_history: dict[str, list[dict]] = {}
```
**影响**: 用户对话历史不持久化，无法支持长期对话上下文、对话分析、审计

**修复方案**:
- 创建 `AgentConversation`, `AgentMessage` 模型
- 使用 PostgreSQL 存储对话历史
- 支持对话搜索和审计查询

---

### 3. benchmark_service.py - 性能基准数据内存化
```python
_benchmarks: dict[str, dict] = {}     # 基准测试结果
_trend_data: list[dict] = []          # 趋势数据（最近100条）
```
**影响**: 性能测试数据不持久化，无法分析长期趋势

**修复方案**:
- 创建 `Benchmark`, `BenchmarkTrend` 模型
- 支持历史趋势查询和对比分析

---

### 4. dag_engine.py - DAG 执行状态内存化
```python
_execution_history: dict[str, list[dict]] = {}  # 执行历史
_checkpoints: dict[str, dict] = {}              # 检查点
```
**影响**: 计算任务 DAG 执行状态不持久化，服务重启后无法恢复任务

**修复方案**:
- 创建 `DagExecution`, `DagCheckpoint` 模型
- 任务失败时可从检查点恢复

---

### 5. cluster_service.py - 集群调度状态内存化
```python
_nodes: dict[str, dict] = {}        # 节点信息
_dispatches: dict[str, dict] = {}   # 调度任务
```
**影响**: 集群节点和调度任务不持久化，无法支持分布式部署

**修复方案**:
- 创建 `ClusterNode`, `NodeDispatch` 模型
- 支持集群状态持久化和跨实例查询

---

### 6. cross_chain_service.py - 跨链交易内存存储
```python
_bridge_transactions: dict[str, dict] = {}  # 跨链交易
_pending_messages: dict[str, dict] = {}    # 待处理消息
```
**影响**: 跨链交易状态不持久化，服务重启后交易丢失

**修复方案**:
- 创建 `CrossChainTransaction`, `CrossChainMessage` 模型
- 支持跨链交易查询和审计

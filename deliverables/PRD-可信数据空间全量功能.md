# 南方能源行业可信数据空间 — 全量功能 PRD

## 1. 项目信息

- **Language**: 中文
- **Programming Language**: FastAPI + React + MUI + PostgreSQL
- **Project Name**: `southern_energy_trusted_data_space`
- **原始需求复述**: 基于14份操作指引全面实现南方能源行业可信数据空间的所有业务功能，当前系统仅有约8%的功能完成度，需要全面实现注册入驻、机构管理、数据资源发布上架、数据资源订阅与交付、数据产品开发、数据产品上架、数据产品订阅与交付、需求发布及承接、服务编排、合约管理、连接器文件库、连接器部署等全量业务功能。

## 2. 产品定义

### 2.1 产品目标

基于国家标准《可信数据空间 技术架构》及行业最佳实践，构建南方能源行业可信数据空间：

1. **构建可信数据流通基础设施**: 实现能源行业数据资源的安全、合规、高效流通，支持数据提供方、使用方、运营方三方协作，打造完整的数据要素市场生态
2. **实现全链路数据产品生命周期管理**: 覆盖从数据接入、元数据发现、目录登记、产品开发、上架发布、订阅交付到合约管理的全流程，支持5种数据产品类型（API/报告/应用/模型/数据集）
3. **建立多方安全管控与合规体系**: 通过连接器架构、数字合约、使用控制、审批工作流等机制，确保数据流通全程可控、可追溯、可审计，满足能源行业数据安全合规要求

## 4. 竞品分析（基于调研）

### 4.1 主要竞品分析

#### 深圳数据交易所
- **业务模式**: 数据产品上架、订阅、交付全流程服务
- **优势**: 交易规则完善、合规审查严格、资金存管安全
- **劣势**: 流程相对复杂、审核周期较长

#### 上海数据交易所
- **业务模式**: 数据产品登记、挂牌、交易三阶段流程
- **优势**: 全球首个数据交易所交易规则体系、权属界定清晰
- **劣势**: 主要面向长三角地区、行业覆盖有限

#### 贵阳大数据交易所
- **业务模式**: 全流程交易服务（市场准入→合规登记→评估定价→服务上架→需求撮合→交易备案→支付结算→数据交付）
- **优势**: 全国第一家大数据交易所、数据要素登记DOID服务平台
- **劣势**: 交易透明度有待提升

#### 华为可信数据空间
- **业务模式**: 连接器架构、管控协议、数字合约
- **优势**: 技术架构完整、符合国家标准、企业级解决方案
- **劣势**: 定制化成本高、生态相对封闭

#### 蚂蚁链可信数据空间
- **业务模式**: 隐私计算、联邦学习、数据沙箱集成
- **优势**: 隐私保护技术领先、多方安全计算能力强
- **劣势**: 主要面向金融行业、能源行业适配不足

#### 浪潮可信数据空间
- **业务模式**: 行业可信数据空间平台、数据链网融合技术体系
- **优势**: 首批通过国家级测试、行业应用场景丰富
- **劣势**: 主要面向政务和通信行业

## 5. 技术架构建议

### 5.1 系统架构

基于现有技术栈（FastAPI + React + MUI + PostgreSQL）和已实现的66个Router + 623个API端点 + 87个SQLAlchemy模型，建议采用以下架构：

1. **微服务化**: 将现有单体应用拆分为独立微服务（用户服务、机构服务、资源服务、产品服务、合约服务等）
2. **事件驱动**: 使用消息队列实现服务间异步通信
3. **缓存优化**: 引入Redis缓存热点数据
4. **搜索增强**: 集成Elasticsearch提升搜索性能
5. **文件存储**: 使用对象存储（如MinIO）管理文件资源

### 5.2 数据库设计优化

基于现有87个SQLAlchemy模型，需要补充以下核心表：
- 机构认证表（机构类型、认证材料、审核状态）
- 连接器配置表（部署信息、数据源配置、同步状态）
- 合约表（合约类型、条款内容、签署状态）
- 审批流程表（流程定义、节点配置、审批记录）
- 审计日志表（操作类型、操作对象、操作时间）

### 5.3 安全设计

1. **身份认证**: 基于现有JWT + SM3 + MFA(TOTP) + SSO + DID + SM2体系
2. **权限控制**: RBAC + ABAC混合权限模型
3. **数据安全**: 传输加密（SM4）、存储加密、数据脱敏
4. **审计追踪**: 全链路操作日志、区块链存证

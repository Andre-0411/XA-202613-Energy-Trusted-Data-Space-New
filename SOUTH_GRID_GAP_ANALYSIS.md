# 南方电网操作指引 vs 我们系统 — 差距分析

## 核心业务流程对标

### 1. 注册入驻流程（01指引）
**南方电网**：邀请码注册 → 机构认证（企业/政府/事业单位）→ 上传营业执照/信用报告 → 管理员审核 → 成员加入
**我们现状**：admin创建用户 → 无邀请码 → 无机构认证材料上传 → 无两级审核
**差距**：
- 缺少邀请码机制
- 缺少机构认证材料上传（营业执照、信用报告、法人身份证）
- 缺少机构认证审核流程
- 缺少成员加入申请+管理员审核

### 2. 机构管理员业务（02指引）
**南方电网**：成员准入审批 → 业务角色分配（12种角色）→ 成员账号管理
**我们现状**：admin直接创建用户 → 角色只有4种 → 无审批流程
**差距**：
- 缺少成员准入审批
- 角色体系需扩展到12种
- 缺少业务角色分配界面

### 3. 数据资源发布上架（03指引）
**南方电网**：连接器部署 → 数据源接入 → 元数据发现 → 摘要分析 → 敏感发现 → 安全等级定级 → 数据目录注册 → 供给通道配置 → 控制协议配置 → 两级审核
**我们现状**：数据源管理 → 资产注册 → 目录发布 → 单级审核
**差距**：
- 缺少元数据自动发现
- 缺少摘要分析和敏感发现
- 缺少安全等级自动定级
- 缺少供给通道配置（数据集/API/隐私计算/密文查询）
- 缺少控制协议配置
- 缺少两级审核（机构管理员+平台运营方）

### 4. 数据资源订阅与交付（04指引）
**南方电网**：数据发现 → 订阅申请 → 审批 → 连接器交付 → 使用控制
**我们现状**：服务申请 → 审批 → 简单交付
**差距**：
- 缺少连接器交付机制
- 缺少使用控制（停止权限、限制使用）
- 缺少数据使用计量

### 5. 数据产品开发（05指引）
**南方电网**：创建项目 → 配置成员 → 配置数据源 → 创建开发任务 → 开发加工测试 → 产品验收
**我们现状**：计算任务创建 → 执行 → 结果查看
**差距**：
- 缺少项目管理（创建/成员/数据源配置）
- 缺少开发任务与项目关联
- 缺少产品验收流程

### 6. 数据产品上架（06指引）
**南方电网**：提交发布申请 → 内容审核 → 合规审核 → 上架
**我们现状**：产品管理页面已有 → 但缺少合规审核
**差距**：
- 缺少合规材料审核
- 缺少上架公示

### 7. 数据产品订阅与交付（07指引）
**南方电网**：产品检索 → 订阅 → 交付审批 → 合同备案 → 使用 → 结算
**我们现状**：产品市场 → 订阅 → 简单结算
**差距**：
- 缺少合同备案
- 缺少使用计量
- 缺少智能合约控制

### 8. 需求发布与承接（08指引）
**南方电网**：需求发布 → 运营方审核 → 需求展示 → 认领承接 → 自动下架
**我们现状**：需求管理页面已有 → 但缺少审核和自动下架
**差距**：
- 缺少需求审核
- 缺少认领后自动下架

### 9. 服务编排（09指引）
**南方电网**：可视化编排数据处理流程 → 拖拽式配置
**我们现状**：DAG编排页面已有 → 需要增强
**差距**：
- 需要增强DAG编排的可视化

### 10. 合约管理（10指引）
**南方电网**：智能合约创建 → 部署 → 使用控制
**我们现状**：合约管理页面已有 → 需要增强
**差距**：
- 需要增强合约与数据使用控制的关联

### 11. 连接器文件库（11指引）
**南方电网**：文件上传 → 文件集管理 → API代理
**我们现状**：连接器文件库页面已有 → 需要增强
**差距**：
- 需要增强文件管理功能
